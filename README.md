This repository contains the full implementation for **Task 1–Task 4** of the DSAI5203 Individual Assignment 2 (Auditory Attention Decoding on DTU‑S2 Subject 2).

## Task 1: EEG Preprocessing and Visualization

### What is implemented

1. Load EEG data and labels from `S2.mat` and map labels to binary `0/1`
2. Keep the first 64 EEG channels (scalp channels)
3. Build MNE objects (`RawArray` for processing, `EpochsArray` for segment-level analysis)
4. Apply preprocessing pipeline:
   - 4–40 Hz FIR band-pass filter
   - 50 Hz notch filter
   - average re-referencing
5. Reshape output back to segment format `(N_segments, 64, N_timepoints)`
6. Generate Task 1(c) visualizations (before/after filtering):
   - centered 10-channel view (before/after filtering)
   - normalized 10-channel view (before/after filtering)
   - single-channel zoomed comparison
7. Compute channel-wise average band power and draw topomaps for:
   - Theta (4–8 Hz)
   - Alpha (8–13 Hz)
   - Beta (13–30 Hz)

### Output of Task 1

- `X_prep`: preprocessed EEG segments
- `y`: binary labels (`0/1`)
- `epochs`, `info`, `sfreq`
- saved figures under `pic/`

## Task 2: SVM Classification (AAD)

### What is implemented

1. Use `X_prep` and `y` from Task 1
2. Build a filter-bank + CSP feature pipeline:
   - multiple band-pass filters
   - CSP per band (log average power)
   - feature concatenation
3. Standardize features on training split
4. Apply feature selection (`SelectKBest` with mutual information)
5. Perform stratified 80/20 train-test split
6. Train and compare four SVM kernels:
   - linear
   - rbf
   - poly
   - sigmoid
7. For each kernel, run a lightweight parameter search on the same split
8. Print progress, timing, best params, and final kernel comparison

### Output of Task 2

- accuracy per kernel
- best kernel and test accuracy
- trained best model object
- fitted scaler and feature selector
- CSP models and band definitions

### Example Result and Brief Analysis

Example result from the current implementation:

- `linear`: test = `0.5667`, val = `0.5521`
- `rbf`: test = `0.5600`, val = `0.5625`
- `poly`: test = `0.5600`, val = `0.5583`
- `sigmoid`: test = `0.5067`, val = `0.5687`
- best kernel on test set: `linear` (`0.5667`)

Interpretation:

- The result is above random chance (`0.50`) for binary AAD, so it is a valid baseline.
- The performance gain is moderate, which is common for short-window (1-second) EEG AAD with traditional SVM pipelines.
- Some kernels (e.g., sigmoid) show higher validation accuracy but lower test accuracy, indicating unstable generalization and possible overfitting.
- Therefore, Task 2 provides a reasonable classical ML reference, while Tasks 3/4 are expected to further improve robustness and performance.

## Task 3: Transformer‑Style EEG Baseline (EEG FM)

### What is implemented

1. Use `X_prep` and `y` from Task 1
2. Standardize features using training split only, then reshape back to `(N, C, T)`
3. Train a Transformer‑encoder baseline (`EEGFMBaseline`):
   - linear input projection
   - multi‑layer Transformer encoder
   - layer norm + linear classifier
4. Track train/test accuracy by epoch and save best checkpoint
5. Plot training curves using `train_utils.plot_training_history`

### Output of Task 3

- `fm_model`, `fm_hist`, `fm_best_acc`
- checkpoint saved to `checkpoints/fm_baseline_best.pth`

## Task 4: Advanced Deep Model (Conv + BiGRU + Attention)

### What is implemented

1. Use the same train/test split and standardization protocol as Task 3
2. Train a stronger model (`AdvancedAADNet`) with:
   - temporal convolution stack (local feature extraction)
   - BiGRU (sequence modeling)
   - multi‑head self‑attention
   - mean pooling + classifier
3. Track train/test accuracy and report best test accuracy

### Output of Task 4

- `adv_model`, `adv_hist`, `adv_best_acc`
